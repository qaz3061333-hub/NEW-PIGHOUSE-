# Price Quote Sandbox KB Manual Creation Plan v1

資料日期：2026-05-24

本文件適用於 NEW-PIGHOUSE- / new-pighouse-pjdh 專案。此專案目前仍是 sandbox / MVP / 內部流程驗證階段，不是正式 LINE 上線階段。

## 一、文件目的

本文件是給使用者手動建立 sandbox Knowledge Base 測試資料用。

本文件不是正式匯入、不是 SQL、不是程式、不是正式 LINE 自動報價，也不是全價目表整理。

本文件的目標是：先選少量代表性價格與加購資料，由使用者在後台 Knowledge Base 手動建立成 active KB，然後用 sandbox Conversation Logs / KB 查詢測試是否能正確命中。

這一步只是在 sandbox 驗證「文章切法、查詢命中、缺資料追問、另計提醒、高風險轉人工」是否合理。正式價格、加價規則、分店限制、現場評估條件、對客提醒，未來都應由 active Knowledge Base 管理，不應寫死在 code、API、prompt 或 Gemini 記憶中。

## 二、為什麼要手動建立少量 KB，而不是一次匯入全部價目表

價格是最容易出錯、也最容易造成客訴的資料。如果一開始就把完整價目表全部匯入，格式只要不適合後台或查詢邏輯，後面會變成大量返工。

先手動建立少量樣本，可以先驗證幾件最重要的事：

- sandbox Conversation Logs / KB 查詢是否能命中正確文章。
- 缺品種、體重、膚質、分店時，系統是否會追問，而不是亂猜。
- 另計項目是否只提醒現場評估，不會自動算完整總價。
- 醫療疑慮、價格爭議、退款、客訴、受傷、流血等高風險情境是否會轉人工。
- 後台 Knowledge Base 的 title / category / content 實際使用方式是否好貼、好讀、好維護。

等少量測試通過後，再考慮擴充更多犬種，或另開任務規劃正式匯入流程。這樣比較容易發現問題，也比較不會把錯誤格式擴散到整包 KB。

## 三、第一批建議手動建立的 sandbox KB 文章

以下 9 篇是第一批建議手動建立的 sandbox KB 文章。這不是正式 KB 清單，也不是完整價目表，只是用來測試不同報價情境的代表性樣本。

### 1. 價格｜馬爾濟斯

用途：小型犬標準報價測試。

測試重點：品種 + 體重 + 洗澡 / 大美容 / 包月報價區間。

這篇適合測最基本的犬種查價流程，例如客人有提供「馬爾濟斯 3kg」時，可以命中正確體重區間；客人只說「馬爾濟斯」但沒說體重時，應該追問體重。

### 2. 價格｜貴賓（手吹）

用途：手吹犬種三個加價級距測試。

測試重點：200 / 250 / 300 加價級距、包月同邏輯、手修加價提醒。

這篇適合測手吹犬種是否能保留三個級距邏輯，以及包月是否跟單次服務使用同樣加價級距。若客人提到手剪、腳柱、屁屁、領巾等手修需求，應提醒手修加價，不應在沒有手修需求時自動加入。

### 3. 價格｜黃金

用途：大型犬與另計規則測試。

測試重點：可先報大美容基礎區間，手修另計需現場評估，不要自動算完整總價。

這篇適合測「另計」不是什麼都不能回答，而是可以先回答可確定的基礎報價，再提醒另計項目需現場評估。

### 4. 價格｜貓咪分類

用途：貓咪不是依犬種，而是依短毛 / 中長毛、長毛、巨型貓種分類。

測試重點：一般貓可報區間，巨型貓種例如緬因貓另計，不自動報價。

這篇適合測貓咪報價不要硬套犬種邏輯。若客人只說貓咪洗澡，應追問毛長類型、體型或品種資訊。

### 5. 加購｜單項基本美容服務

用途：剪指甲、磨指甲、清耳朵、拔耳毛等單做項目。

測試重點：洗澡 / 大美容已包含，不重複加收；單做時才報單項價格；嚴重耳況建議就醫。

這篇適合測加購項目不要被自動加進洗澡或大美容報價，也適合測「耳朵流膿」這類醫療疑慮不能直接報清耳朵價格。

### 6. 加購｜草本泥與火山泥

用途：草本泥、火山泥按體重與膚質報價。

測試重點：20kg 以下可報，20kg 以上不自動報價，請致電或轉人工。

這篇適合測系統是否會追問體重與膚質，也測 20kg 以上是否不自行外推價格。

### 7. 規則｜體外蟲與環境清潔

用途：跳蚤 / 體外蟲 / 環境清潔規則。

測試重點：活體成蟲 +150 除蚤洗劑；超過 10 隻成蟲收環境清潔費 300～800；蟲量過多可退貨並收已完成項目費用。

這篇適合測共通規則命中，也測系統是否能在不保證一定承接的前提下，說明現場可能處理方式。

### 8. 住宿｜住宿外加項目

用途：散步、櫃內插座、睡墊租用 / 購買。

測試重點：散步需自備不易掙脫胸背牽繩；過強攻擊性不可加購；睡墊只限中和店。

這篇適合測住宿外加項目與分店限制，尤其是睡墊租用 / 購買不可直接承諾非中和店也提供。

### 9. 規則｜報價共通提醒與高風險邊界

用途：共同提醒與不能自動完整回答的情境。

測試重點：價格爭議、退款、客訴、受傷、流血、醫療疑慮，不應用價格 KB 自動完整回答。

這篇是安全邊界文章。它不是拿來報價，而是讓 sandbox 測試能確認高風險情境會走人工流程。

## 四、每篇 KB 建議欄位格式

建議每篇手動建立的 KB 文章，都用以下固定格式整理。這樣使用者貼到後台時比較不會漏欄位，之後測試也比較容易看出是哪個欄位被命中。

```text
標題：
類型：
適用對象：
主要內容：
可自動報價：
需追問資訊：
需現場評估情況：
不可自動完整回答情況：
對客提醒：
測試問題：
```

如果目前後台 KB 只有 `title` / `content` / `category` 這類簡單欄位，不需要新增 table，也不需要改 schema。可以先把上面的欄位全部放在 `content` 文字中，`title` 放文章標題，`category` 放文章類型即可。

## 六、手動建立順序

建議不要一次建立 9 篇。先用最基本、最容易測的資料驗證 KB 查詢，再逐步加入複雜規則。

第一批先建 3 篇：

1. 價格｜馬爾濟斯
2. 加購｜單項基本美容服務
3. 規則｜報價共通提醒與高風險邊界

通過後再建：

4. 價格｜貴賓（手吹）
5. 價格｜黃金
6. 價格｜貓咪分類

最後再建：

7. 加購｜草本泥與火山泥
8. 規則｜體外蟲與環境清潔
9. 住宿｜住宿外加項目

理由：先用基本犬種、基本單項服務、高風險邊界確認查詢與安全流程，再逐步測手吹級距、大型犬另計、貓咪分類、20kg 以上不外推、體外蟲規則與住宿分店限制。

## 七、sandbox 測試問題

以下題目用來在 sandbox Conversation Logs / KB 查詢測試。重點不是讓文案完全一樣，而是確認命中、追問、另計與轉人工方向正確。

| 測試題 | 期望結果 |
| --- | --- |
| 馬爾濟斯 3kg 洗澡多少？ | 命中馬爾濟斯 KB，報洗澡區間。 |
| 馬爾濟斯洗澡多少？ | 缺體重，應追問體重，不猜價格。 |
| 只剪指甲多少？ | 命中單項基本美容服務 KB，報 100 起並提醒激動 / 攻擊風險可能加價或無法承接。 |
| 我家狗耳朵流膿，可以清耳嗎？ | 不要報清耳朵 50，應建議就醫或人工處理。 |
| 黃金要手剪多少？ | 先報黃金大美容可確定區間，再提醒手修費用另計、需現場評估。 |
| 緬因貓洗澡多少？ | 命中貓咪分類，說明巨型貓種另計，不自動報價。 |
| 火山泥 25kg 多少？ | 不要外推價格，提醒 20kg 以上請致電或轉人工。 |
| 狗狗有跳蚤怎麼收費？ | 命中體外蟲規則，說明活體成蟲 +150，超過 10 隻環境清潔費 300～800，蟲量過多可能退貨並收已完成費用。 |
| 住宿可以加散步嗎？ | 命中住宿外加，說明可加購 100 / 10 分鐘，但需自備不易掙脫胸背牽繩，過強攻擊性不可加購。 |
| 美容完流血怎麼辦？ | 不得用價格 KB 自動回答，應走高風險人工流程。 |

## 八、通過標準

這批 sandbox KB 測試通過，至少應符合以下標準：

- 能正確命中相關 KB。
- 缺品種 / 體重 / 膚質 / 分店時會追問，不亂猜。
- 另計項目不自動算完整總價。
- 醫療疑慮不由系統判斷，應建議就醫或人工處理。
- 價格爭議、退款、客訴、受傷、流血會走人工流程。
- inactive KB 不應被用來回答。
- KB 無資料時不可亂編，應建立 Knowledge Gap 或轉人工。
- 不把整包 KB 丟給 Gemini，只取少量相關 KB 產生草稿。
