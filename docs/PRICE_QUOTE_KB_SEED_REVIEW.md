# Price Quote KB Seed Review v1

資料日期：2026-05-23

本文件適用於 NEW-PIGHOUSE- / new-pighouse-pjdh 專案。此專案目前仍是沙盒 / MVP / 內部流程驗證階段，不是正式 LINE 上線階段。

## 1. 文件目的

本文件用來審核 `docs/PRICE_QUOTE_KB_SEED_DRAFT.md` 這份 Price Quote KB Seed Draft v1 的格式、可維護性、未來系統查詢適配性，以及目前仍需要保留的風險邊界。

本文件只做 review，不改任何價格資料，不匯入資料庫，不建立正式 Knowledge Base 文章，也不代表正式 LINE 可以自動報價。這份 review 的重點是先確認第一批 seed draft 是否適合未來繼續往價格 KB 方向整理。

## 2. 本次審核範圍

本次只審核 seed draft 中第一批樣本：

- 馬爾濟斯
- 貴賓（手吹）
- 法鬥
- 黃金
- 貓咪分類

本次不審全犬種，不審完整加購項目，不審住宿外加，不審正式自動報價功能。

本次也不擴充價目表、不修正價格、不匯入資料庫、不改程式，只檢查目前草案格式是否清楚、風險是否有標示，以及哪些規則需要使用者再確認。

## 3. 審核標準

### A. 格式是否好懂

- 使用者能不能直接看懂每個體重區間。
- 店員能不能知道洗澡、大美容、包月各自報多少。
- 是否有清楚標示另計、手修、人工確認。
- 是否能一眼看出哪些內容可以先給初步區間，哪些內容不能自動報價。

### B. 格式是否好維護

- 未來價格變動時，是否容易改。
- 是否避免一大段散文，改成固定欄位。
- 是否能看出價格區間怎麼來。
- 是否能把品種、體重、服務項目、基礎價、加價上限、對客報價區間分開維護。

### C. 是否適合未來系統查詢

- 品種 / 類型是否清楚。
- 體重區間是否清楚。
- 洗澡、大美容、包月是否分開。
- 對客報價區間是否已直接列出。
- 另計與人工確認是否清楚。
- 未來若系統查詢 KB，是否可以避免把單一散文段落誤解成完整價格規則。

### D. 是否符合安全邊界

- 不讓系統猜價。
- 不自動處理另計。
- 不自動處理價格爭議。
- 不自動處理退款、客訴、受傷、流血、醫療疑慮。
- 不把草案當正式 LINE 回覆來源。
- 不把價格草案直接當成已通過人工確認的正式 KB。

## 4. 整體審核結論

第一批格式整體可讀性良好。每個樣本都有品種 / 類型、體重區間、洗澡、大美容、包月、對客報價區間等資訊，店員或使用者可以看出每個級距對應的基本報價方式。

這種格式比一整段自然語言更適合人工維護與未來系統查詢。因為價格、體重、服務類型、加價上限與注意事項都有被拆開，未來如果要調整價格，也比較不容易改錯整段文字。

但目前仍是草案，不能直接匯入正式 KB。原因是仍有幾個關鍵報價規則需要使用者先確認，尤其是貴賓包月第三級距、另計、手修、加購項目是否適用，以及法鬥 / 鬥牛犬類、貓咪巨型貓種等特殊規則是否能一律照目前文字處理。

因此，本 review 的結論是：第一批 seed draft 可以作為「價格 KB 格式驗證樣本」，但還不能作為正式 LINE 自動報價資料來源。

## 5. 各樣本審核

### A. 馬爾濟斯

馬爾濟斯樣本格式清楚。三個體重區間都有列出，分別是 3kg 以下、3.1～4.5kg、4.5kg 以上。

每個區間都有列出洗澡基礎價、大美容基礎價、包月基礎價，也都有直接列出洗澡、大美容、包月的對客報價區間。這對人工檢查和未來系統查詢都很直覺。

手修加價有保留，且共同提醒中也有寫到如果需要腳柱、屁屁、領巾、全身手剪等手修，需另外加入手修加價。這有助於避免系統把手修自動算進一般洗澡或大美容報價。

馬爾濟斯目前可作為小型犬標準範例。以格式來看，沒有明顯格式問題。

### B. 貴賓（手吹）

貴賓（手吹）樣本格式可用。三個體重區間都有列出，並且有處理不同級距的毛長 / 激動加價上限，分別保留 200、250、300 的加價上限。

手修加價有保留，且文件有明確說明手修不是一般報價預設項目。只有客人提到腳柱、屁屁、領巾、全身手剪、手剪造型等需求時，才應該納入手修加價。

目前有標記「貴賓包月第三級距」需要人工確認，這點很重要。因為 seed draft 中第三級距使用 L 包月 2500 並算出 2500～2800，但原價目表下方範例曾出現「6-8.9 即報 2300～2550」的描述，兩者可能有級距對應或範例文字需要核對。

因此，不能把貴賓包月第三級距當正式定稿。後續應由使用者確認到底依 L 包月 2500 計算，還是依表下方範例 2300～2550。

### C. 法鬥

法鬥樣本格式清楚。三個體重區間都有列出，洗澡、大美容、包月基礎價與對客報價區間也都有列出。

手修欄位為無，這點有保留。這很重要，因為法鬥不應被系統自動套用其他犬種的手修加價規則。

包月特殊贈送項目也有保留，包括極致保濕足部護理 / 鼻頭護理 / 藥浴 3 合 1 療程，以及韓國低弱敏皮毛養護或局部潔膚油。這類內容不是單純價格，但對店員確認包月內容有幫助。

法鬥可作為短毛犬 / 鬥牛犬類範例。後續需確認「鬥牛犬類包月多送洗劑或潔膚油」是否適用所有鬥牛犬類，或只適用表中部分犬種。

### D. 黃金

黃金樣本格式清楚。三個大型犬體重區間都有列出，洗澡、大美容、包月的基礎價與對客報價區間也都有列出。

大型犬體型 / 毛長 / 激動加價較高，有保留為 200～600。這可以提醒未來系統或店員，大型犬價格受現場狀況影響較大，不應用小型犬的加價邏輯處理。

手修另計有保留。大型犬不應自動處理手修或特殊造型，尤其是需要看毛量、廢毛、打結、配合度或現場狀況時，應轉人工確認。

黃金可作為大型犬 / 另計規則範例。後續需確認大型犬嚴重廢毛、打結、手修是否一律轉人工，而不是只用固定加價上限自動算出完整價格。

### E. 貓咪分類

貓咪分類的整理方向合理。貓咪不是依犬種，而是依短毛 / 中長毛、長毛、巨型貓種分類，這和犬種查表邏輯不同，有明確區分。

短毛～中長毛貓、長毛貓已可直接列報價區間。短毛～中長毛貓目前列出洗澡 600～1300、大美容 1400～2100；長毛貓目前列出洗澡 800～1700、大美容 1600～2300。

巨型貓種為另計，應轉人工或現場評估，不應自動報價。

若客人只說「貓咪洗澡」，系統或店員應追問毛長類型、體型或品種資訊。不能直接用短毛貓、長毛貓或巨型貓種任一分類猜價。

## 6. 目前不可進入正式 KB 的原因

目前 seed draft 不能直接成為正式 KB，原因是：

- 仍有待使用者確認問題。
- 還沒有人工確認所有價格是否與原價目表完全一致。
- 還沒有整理加購項目。
- 還沒有整理全犬種。
- 還沒有測試 Conversation Logs 能否正確命中。
- 還沒有測試缺品種 / 缺體重時是否正確追問。
- 還沒有測試另計 / 手修 / 價格爭議是否正確轉人工。

換句話說，目前這份 seed draft 比較像格式樣本與人工 review 草案，不是正式可查詢的完整價格 KB。

## 7. 待使用者確認清單

1. 加價項目表中的剪指甲、磨指甲、拔耳毛等，是否只適用於單做或住宿外加，而不是洗澡 / 大美容內再加一次？
2. 「另計」是否一律代表不可自動報價，需轉人工或現場評估？
3. 貴賓包月第三級距是否依表格 L 包月 2500 計算，還是依下方範例 2300～2550？
4. 手修完整報價是否未來要計算成：大美容基礎價 + 毛長/激動加價 + 手修加價？
5. 嚴重打結、廢毛、體外蟲是否只提醒另計，還是未來要補明確加價範圍？
6. 法鬥 / 鬥牛犬類包月多送洗劑或潔膚油的規則，是否適用所有鬥牛犬類？
7. 貓咪巨型貓種是否一律另計，不自動報價？

## 8. 下一步建議

下一步不要直接擴大全犬種。建議先做：

Price Quote KB User Confirmation v1

目的：把上述待確認問題整理成一份使用者可直接回答的確認清單。

等使用者回答後，再決定是否：

- 修正 seed draft
- 整理加購項目
- 擴充更多犬種
- 或開始做手動 KB 建立清單

這樣會比直接把全犬種全部整理出來更安全，因為如果另計、手修、加購項目或貴賓級距規則還沒確認，擴充越多資料，後面要修正的成本就越高。
