## 1. 文件目的

本文件是第一批價格 Knowledge Base 文字草案，用來把少量代表性犬種與貓咪分類整理成固定格式，讓使用者先人工確認格式是否好懂、好維護，也是否適合未來系統查詢。

本文件只驗證文字格式與資料整理方式，不匯入資料庫，不建立正式 Knowledge Base 文章，不代表正式 LINE 可以自動報價。

本文件不是正式 KB 匯入檔，不是 SQL，不是程式，也不是完整價目表整理。

## 2. 共用報價邏輯摘要

客人報價前，至少要知道：品種 / 類型、體重。

犬種只是查表條件，不是每個犬種各自一套不同邏輯。報價邏輯是共用的，差異只在查表資料，例如犬種、體重區間、基礎價格、加價上限、手修備註與注意事項。

共用邏輯是：

1. 查到品種或類型 + 體重區間。
2. 找到洗澡 / 大美容 / 包月價格。
3. 加上該區間的體型 / 長毛 / 激動加價上限。
4. 對客回報價格區間。

對客報價應回區間，不回單一死價格。

一般公式：

```text
洗澡對客報價 = 洗澡基礎價 ～ 洗澡基礎價 + 體型/長毛/激動加價上限
大美容對客報價 = 大美容基礎價 ～ 大美容基礎價 + 體型/長毛/激動加價上限
包月對客報價 = 包月基礎價 ～ 包月基礎價 + 體型/長毛/激動加價上限
```

如果資料中標示「另計」或需要現場評估，就不應自動算出完整價格，應轉人工確認或保留人工確認提示。

## 3. 第一批樣本選擇理由

A. 馬爾濟斯

理由：小型犬標準範例，價格與加價最直覺，適合驗證基本格式。

B. 貴賓（手吹）

理由：有不同加價級距，也有手修加價，適合驗證格式是否能保留加價欄位、手修公式與人工確認備註。

C. 法鬥

理由：短毛犬、手修為無、包月內容有特殊贈送項目，適合驗證「不可自動加入手修」與特殊備註保存方式。

D. 黃金

理由：大型犬、體型加價高、手修另計，適合驗證人工確認規則與大型犬現場評估提醒。

E. 貓咪分類

理由：貓咪不是依犬種，而是依短毛 / 中長毛、長毛、巨型貓種分類，適合驗證非犬種資料的整理方式。

## 9. 共通報價提醒

未來每篇價格 KB 都應保留以下共同提醒：

- 實際價格會依體型、毛長、激動或攻擊程度調整。
- 若有嚴重打結、廢毛、體外蟲，需現場評估另計。
- 若有拆除過多打結、深層清潔廢毛，將視情況額外收取費用。
- 若有手修、特殊造型或加購項目，會另外計算。
- 報價以現場評估與最新 Knowledge Base 為準。
- 有提及草本泥、火山泥的客人，需提前 10 分鐘到現場。
- 需要現場了解皮毛狀況後，才能給予加價建議，例如升級洗劑或特定部位潔膚油。
