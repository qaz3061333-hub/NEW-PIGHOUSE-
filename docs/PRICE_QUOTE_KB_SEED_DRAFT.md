## 1. 文件目的

本文件是第一批價格 Knowledge Base 文字草案，用來把少量代表性犬種與貓咪分類整理成固定格式，讓使用者先人工確認格式是否好懂、好維護，也是否適合未來系統查詢。

本文件只驗證文字格式與資料整理方式，不匯入資料庫，不建立正式 Knowledge Base 文章，不代表正式 LINE 可以自動報價。

本文件不是正式 KB 匯入檔，不是 SQL，不是程式，也不是完整價目表整理。

## 2. 共用報價邏輯摘要

客人報價前，至少要知道：品種 / 類型、體重。

犬種只是查表條件，不是每個犬種各自一套不同邏輯。報價邏輯是共用的，差異只在查表資料，例如犬種、體重區間、基礎價格、加價上限、手修備註與注意事項。

共用邏輯是：

1. 查到品種或類型 + 體重區間。
2. 找到洗澡 / 大美容 / 包月價格。
3. 加上該區間的體型 / 長毛 / 激動加價上限。
4. 對客回報價格區間。

對客報價應回區間，不回單一死價格。

一般公式：

```text
洗澡對客報價 = 洗澡基礎價 ～ 洗澡基礎價 + 體型/長毛/激動加價上限
大美容對客報價 = 大美容基礎價 ～ 大美容基礎價 + 體型/長毛/激動加價上限
包月對客報價 = 包月基礎價 ～ 包月基礎價 + 體型/長毛/激動加價上限
```

「另計」代表不可自動算出完整總價，但不是一律建立人工任務。系統仍可回覆已能確定的基礎價格，再補充另計項目需依現場體型、毛長、配合度等情況現場報價。

例如黃金獵犬客人問大美容加手剪，系統可先報大美容費用區間，再補充：「手修費用另計，需依現場體型、毛長、配合度等情況現場報價。」這種情況不一定要建立人工任務，只要清楚提醒現場另計即可。

但如果涉及客訴、價格爭議、退款、受傷、流血、醫療疑慮，仍需走高風險人工流程。

## 3. 洗澡與大美容內含基本美容項目

洗澡與大美容都已包含以下基本美容項目：

- 修剪指甲及磨圓
- 修剪腳緣
- 修剃腳底毛
- 修剃肛門毛
- 修剃肚子毛
- 修剃生殖器毛
- 擠肛門腺
- 視情況修開眼周雜毛
- 雜毛修整、廢毛梳整
- 耳朵清潔，包含拔 / 剃耳毛、清耳液清潔耳朵內部

如果客人有特別不要做的基本項目，需主動提出。

如果客人沒有做洗澡或大美容，只單做某項服務，才使用單次 / 單項計費。

系統未來不應在洗澡 / 大美容報價中重複加上這些基本項目費用。

## 4. 手修加價規則

手修不是一般洗澡 / 大美容的預設項目。

只有客人提到全身手修、部分手修、腳柱、屁屁、領巾、手剪造型等需求時，才加入手修邏輯。

手修有分全身手修與部分手修，但都落在手修加價區間內。不要把手修寫成單一固定價格。

手修完整報價方向是：

```text
大美容基礎價 + 毛長 / 激動加價 + 手修加價
```

若手修標示「另計」，則先報大美容可確定區間，再提醒手修現場另計。

## 5. 體外蟲 / 廢毛 / 打結規則

體外蟲：

- 只要發現活體成蟲，一律 +150 改為除蚤洗劑。
- 若超過 10 隻成蟲，會收環境清潔費，最低 300、最高 800，依數量與狀況評估。
- 若蟲量過多，可以退貨，並收取已完成項目之費用。

廢毛與打結：

- 廢毛與打結差距非常大，是否好拆、量多量少都會影響時間與費用。
- 不提供固定加價範圍。
- 只提醒現場評估另計。

## 6. 包月贈送規則

所有品種包月中若有寫「韓國低弱敏皮毛養護」，除非明確寫出數量，否則預設只送一次。

潔膚油為兩處內。

不要把「鬥牛犬類包月多送洗劑或潔膚油」寫成無限制或過度承諾。若某犬種或方案有明確寫數量，才依該數量記錄。

## 7. 第一批樣本選擇理由

A. 馬爾濟斯

理由：小型犬標準範例，價格與加價最直覺，適合驗證基本格式。

B. 貴賓（手吹）

理由：有不同加價級距，也有手修加價，適合驗證格式是否能保留加價欄位、手修公式與後續人工核對備註。

C. 法鬥

理由：短毛犬、手修為無、包月內容有特殊贈送項目，適合驗證「不自動加入手修」與特殊備註保存方式。

D. 黃金

理由：大型犬、體型加價高、手修另計，適合驗證「可先報基礎價格 + 另計提醒」的處理方式。

E. 貓咪分類

理由：貓咪不是依犬種，而是依短毛 / 中長毛、長毛、巨型貓種分類，適合驗證非犬種資料的整理方式。
